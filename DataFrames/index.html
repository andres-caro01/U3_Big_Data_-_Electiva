DataFrames y Datasets

Spark SQL introduce el concepto de DataFrame, una colección distribuida de datos organizada en columnas con nombre. Es conceptualmente equivalente a una tabla en una base de datos relacional, pero con optimizaciones enriquecidas. Los DataFrames pueden ser construidos a partir de una amplia variedad de fuentes (archivos JSON, Parquet, tablas de Hive, RDBMS externos). Los Datasets son una extensión de la API de DataFrame que proporciona seguridad de tipos en tiempo de compilación para lenguajes como Scala y Java (Chambers & Zaharia, 2018).

Componente Catalyst Optimizer

Es un optimizador extensible (Catalyst), corazón del Spark SQL. Al ejecutar una consulta SQL o alguna operación de un DataFrame, este componente convierte o transforma un plan lógico, optimizado, aplicando un conjunto de reglas, generando múltiples planes físicos que finalmente, revisa y opta por el plan con más eficiencia, desde el punto de vista de costos de ejecución en el clúster Spark (Chambers & Zaharia, 2018).

Módulo de interfaz unificada

El SQL de Spark ofrece esta interfaz para mezclar consultas SQL con transformaciones de DataFrames, todo en el mismo programa. Esto permite más potencia y flexibilidad.
