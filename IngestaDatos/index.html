Ingesta en Batch

Este enfoque se centra en mover grandes volúmenes de datos a intervalos programados (por ejemplo, cada hora o cada noche). Es ideal para datos que no requieren un análisis inmediato y para la carga inicial de sistemas de Big Data.
×

Apache Sqoop

Herramienta diseñada específicamente para transferir eficientemente datos en bloque entre bases de datos relacionales (como MySQL, Oracle, PostgreSQL) y el ecosistema Hadoop (principalmente HDFS y Hive). Sqoop funciona generando trabajos MapReduce que se conectan a la base de datos relacional a través de un conector JDBC, leen los datos de una tabla y los escriben en HDFS en formatos paralelizables como Avro o Parquet. Su principal ventaja es la simplicidad y la optimización para transferencias masivas desde fuentes estructuradas (White, 2015).

Apache Flume

Servicio distribuido de alta disponibilidad, confiable que recopila, integra y mueve grandes volúmenes de datos (por ejemplo, logs). Posee una arquitectura de agentes compuestos de tres elementos: Fuente o source, Canal o channel, y Sumidero o sink. Ideal para la obtención o ingesta de datos propios de eventos semiestructurados de generación continua, por ejemplo logs (White, 2015).

Ingesta en Streaming

Este enfoque se ocupa de datos que se generan continuamente y que necesitan ser procesados en tiempo real o casi real.
