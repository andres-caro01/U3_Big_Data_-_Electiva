Glosario de la unidad

Análisis Exploratorio de Datos (EDA)

Es el proceso que permite analizar grupos de datos para identificar las principales características, por lo general de forma gráfica. Permite descubrir anomalías y patrones.

Clúster de cómputo

Un conjunto de ordenadores (nodos) interconectados que trabajan juntos como un único sistema para realizar tareas de procesamiento a gran escala.

Data Lake (Lago de Datos)

Es un almacenamiento centralizado o repositorio para almacenar grandes cantidades de datos con cualquier tipo de estructura en formato nativo (estructurados, semiestructurados y no estructurados).

Data Warehouse (Almacén de datos)

Sistema utilizado para almacenar datos actuales e históricos ya estructurados. Se utiliza para optimizar consultas analíticas complejas. Este sistema se utiliza para generar informes.

ETL Extracción - Transformación y Carga (Load)

Identifica el proceso de extracción de los datos se sus fuentes origen, posteriormente su transformación a formato compatibles con la demás información y adecuados, según su contexto y finalmente cargarlos en el destino definido, para este caso un Data Warehouse o bodega de datos.

IaaS - Infrastructure as a Service

Corresponde a una Infraestructura como Servicio. Es un modelo de computación en la nube. Esta provee servicios y recursos de computación virtualizados, como, por ejemplo: servidores, redes y almacenamiento).

Ingesta de datos

Se define ingesta como el proceso que mueve los datos desde cualquier fuente de datos al destino o sistema donde se almacenan, para luego ser accedidos, analizados y hacer uso de ellos.

Machine Learning - Aprendizaje Automático

Es uno de los campos de inteligencia artificial, basado en el uso de algoritmos, que les permiten el aprendizaje a los sistemas y puedan a partir de la experiencia obtenida de los datos, mejorar predicciones.

PaaS - Platform as a Service

Corresponde a plataforma como servicio. Es un modelo de computación en la nube que provee el servicio el cual permite el desarrollo, ejecución y gestión de las aplicaciones, garantizando su funcionamiento y ahorrándose la complejidad de mantener una infraestructura en sitio.

Pipeline de datos

Se define como la serie de pasos en los que se dividen los procesos de datos donde la salida de uno de los procesos o paso pasa a ser la entrada del siguiente paso. Todo garantizando el flujo de datos dentro del proceso.

SaaS - Software as a Service

Es lo que se llama software como servicio. Cuando un proveedor ofrece el uso de un software que aloja una aplicación y queda disponible para los clientes a través de los servicios de red.

Serverless - Sin Servidor

Modelo de servicio en la nube, donde la ejecución y gestión se asigna dinámicamente sin el uso de servidores.
